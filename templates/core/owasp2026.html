Draft Categories

A01:2026 (Expected)
Broken Access Control

Retained from 2021, expanded to cover BOLA and BFLA in REST/GraphQL APIs. New emphasis on mass assignment and feature-level access control failures in microservices.

API Security Failures

Breaking out from A01:2021 — dedicated category for REST, GraphQL, and gRPC vulnerabilities. Covers BOLA, broken function-level auth, excessive data exposure, and missing rate limiting.

AI / ML Security Risks

Prompt injection, model inversion attacks, training data poisoning, and insecure LLM deployment in web applications. Driven by LLM proliferation since 2023.

Dependency confusion, typosquatting on package registries (npm, pip), malicious packages, and compromised CI/CD pipelines. SolarWinds-pattern applied at the application layer.
